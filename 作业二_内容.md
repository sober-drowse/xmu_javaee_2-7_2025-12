

作业二：售后审核通过 
一、实验目的 
1、掌握微服务体系结构 
2、掌握Docker部署文件的编写方法。 
2、掌握通过openfeign调用微服务的方法。 
二、实验环境 
1、服务器A: Ubuntu 18.04 服务器 2核4G内存虚拟机一台，命令行界面，部署售后模块 
2、服务器B：Ubuntu 18.04 服务器 2核2G内存虚拟机一台，命令行界面，部署服务模块 
3、服务器C：Ubuntu 18.04 服务器 2核2G内存虚拟机一台，命令行界面， MySQL 9.0 
三、实验内容及要求 
要  求  实  现  售  后  模  块  中  的  审  核  售  后  单  的API
（https://qbcgzddsjf.apifox.cn/361366645e0）中维修类型售后的API和服务模块中创建
服务单的API（https://qbcgzddsjf.apifox.cn/361366833e0）。要求在审核同意维修类售
后单时，通过openfeign调用服务模块的api创建服务单。需在售后模块和服务模块中的对
应api中输出日志。 
API API描述  
审核售后单 PUT /shops/{shopId}/aftersales/{id}/confirm 
 
 
创建服务单 POST /internal/shops/{shopId}/aftersales/{id}/serviceorders  
 
四、实验检查要求 
1、现场检查数据库自行设计。实现代码。  
2、三台服务器采用Docker SWARM部署，现场检查部署文件和服务器的配置情况。 
3、采用Idea Client测试服务器上PUT 
/shops/{shopId}/aftersales/{id}/confirm，查看服务器上售后模块和服务模块的日志，
以及售后单数据表和服务单数据表的数据变化情况。
